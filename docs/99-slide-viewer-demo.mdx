# Slide Viewer Demo

This page demonstrates how to embed Google Slides presentations in your Docusaurus documentation.

## Basic Usage

You can embed a single slide presentation directly:

<SlideViewer 
  url="https://docs.google.com/presentation/d/1YZJRVyoYe1sAtQXCcTxdjpaIQ84-L57842vydaBjcZk/edit"
  title="Sample Presentation"
  caption="This is a demo presentation"
  height="500px"
/>

## Starting at a Specific Slide

You can start the presentation at a specific slide:

<SlideViewer 
  url="https://docs.google.com/presentation/d/1YZJRVyoYe1sAtQXCcTxdjpaIQ84-L57842vydaBjcZk/edit"
  slideNumber={3}
  title="Starting at Slide 3"
  caption="This presentation starts at the third slide"
  height="500px"
/>
## Multiple Slides from Frontmatter

You can also define multiple slides in your page's frontmatter and display them all:

<MultiSlideViewer slides={frontMatter.slides} />

## Code Example

Here's how to use the SlideViewer component in your MDX files:

```jsx
import SlideViewer from '@site/src/components/SlideViewer';

// Basic usage
<SlideViewer 
  url="https://docs.google.com/presentation/d/YOUR_ID/edit"
  title="My Workshop Introduction"
  caption="Getting started with our workshop"
  height="600px"
  autoplay={true}
  loop={false}
  delayMs={5000}
  notesLink="/docs/speaker-notes"
/>

// Start at a specific slide number (1-based)
<SlideViewer 
  url="https://docs.google.com/presentation/d/YOUR_ID/edit"
  slideNumber={5}  // Starts at slide 5
  title="Architecture Overview"
/>

// Start at a specific slide using slide ID
<SlideViewer 
  url="https://docs.google.com/presentation/d/YOUR_ID/edit"
  slideId="g35df0c0f3b1_0_12007"  // Specific slide ID
  title="Key Concepts"
/>

// Or include the slide ID in the URL (will be auto-extracted)
<SlideViewer 
  url="https://docs.google.com/presentation/d/YOUR_ID/edit#slide=id.g35df0c0f3b1_0_12007"
  title="Direct Link to Slide"
/>
```

## Frontmatter Configuration

You can configure slides in your MDX frontmatter:

```yaml
---
slides:
  - title: "Presentation Title"
    url: "https://docs.google.com/presentation/d/YOUR_ID/edit"
    caption: "Description of the presentation"
    autoplay: true
    loop: true
    notesLink: "/docs/notes"
---
```

## Features

- **Automatic URL Conversion**: Handles various Google Slides URL formats
- **Responsive Design**: Adapts to different screen sizes
- **Dark Mode Support**: Automatically adjusts to theme
- **Loading States**: Shows spinner while loading
- **Enhanced Error Handling**: 
  - Detects permission issues (ðŸ”’)
  - Handles 404/deleted presentations
  - Shows specific error messages
  - Timeout detection with retry option
  - Invalid URL format detection
- **Pre-flight Checks**: Optional permission checking before load
- **Customizable**: Height, width, captions, and more

## Supported URL Formats

The component supports these Google Slides URL formats:
- `https://docs.google.com/presentation/d/{id}/edit`
- `https://docs.google.com/presentation/d/{id}/view`
- `https://drive.google.com/file/d/{id}/view`
- Already embedded URLs (`/embed`)

## Error Handling

The SlideViewer component provides comprehensive error handling:

### Error Types
1. **Permission Errors** (ðŸ”’): Shown when the presentation requires authentication
2. **Invalid URL**: Displayed for malformed Google Slides URLs  
3. **Timeout Errors**: Triggered after 10 seconds (configurable)
4. **Load Errors**: Generic errors during iframe loading

### Configuration Options
```jsx
<SlideViewer 
  url="..."
  loadTimeout={15000}  // Custom timeout in milliseconds
  fallbackContent={    // Custom error content
    <div>
      <p>Custom error message</p>
      <a href="/alternative">Alternative link</a>
    </div>
  }
/>
```

### Using SlideViewerWithCheck
For critical presentations, use the enhanced version that pre-validates access:

```jsx
import SlideViewerWithCheck from '@site/src/components/SlideViewer/SlideViewerWithCheck';

<SlideViewerWithCheck 
  url="https://docs.google.com/presentation/d/YOUR_ID/edit"
  title="Important Presentation"
/>
```

This component:
- Attempts to validate presentation access before rendering
- Provides more specific error messages for permission issues  
- Shows a "Request Access" button for private presentations

## Finding Slide IDs

To get a specific slide ID from Google Slides:

1. Open your presentation in Google Slides
2. Navigate to the slide you want
3. Look at the URL - it will contain `#slide=id.XXXXXXX`
4. Copy the ID part after `id.` (e.g., `g35df0c0f3b1_0_12007`)

Example URL:
```
https://docs.google.com/presentation/d/ABC123/edit#slide=id.g35df0c0f3b1_0_12007
```

You can then use this ID directly:
```jsx
<SlideViewer 
  url="https://docs.google.com/presentation/d/ABC123/edit"
  slideId="g35df0c0f3b1_0_12007"
/>
```

Or just pass the full URL with the slide hash:
```jsx
<SlideViewer 
  url="https://docs.google.com/presentation/d/ABC123/edit#slide=id.g35df0c0f3b1_0_12007"
/>
```
